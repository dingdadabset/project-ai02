# 割草游戏 - 需求文档 (Requirements)

## 1. 项目概述

### 1.1 游戏名称
暂定：**草原风暴 (Grass Storm)**

### 1.2 游戏类型
割草类动作游戏（Horde Survival / Auto-Battle）

### 1.3 参考游戏
- 弓箭传说 (Archero)
- 吸血鬼幸存者 (Vampire Survivors)
- 暗黑破坏神：不朽

### 1.4 目标平台
- **主要平台**：移动端（iOS / Android）
- **次要平台**：微信小游戏、抖音小游戏（后期扩展）

### 1.5 技术选型
- **游戏引擎**：Cocos Creator 3.8+（或 COCOS 4）
- **开发语言**：TypeScript
- **物理引擎**：Cocos 内置物理系统（Bullet）
- **UI 框架**：Cocos UI 组件
- **音频**：Cocos AudioEngine
- **网络**：WebSocket（后期联机/排行榜）
- **构建工具**：Cocos Creator 内置构建系统

## 2. 核心玩法

### 2.1 游戏循环 (Core Loop)

```
进入关卡 → 角色自动攻击 → 移动躲避 → 击杀怪物获得经验
    → 升级选择技能 → 清除更多怪物 → Boss战 → 通关/死亡
        → 获得金币/装备 → 强化角色 → 进入下一关
```

### 2.2 角色系统

| 属性 | 说明 |
|------|------|
| 生命值 (HP) | 角色血量，归零则死亡 |
| 攻击力 (ATK) | 基础伤害值 |
| 攻击速度 (APS) | 每秒攻击次数 |
| 移动速度 (SPD) | 角色移动速度 |
| 暴击率 (CRIT) | 暴击概率 |
| 暴击伤害 (CRIT_DMG) | 暴击倍率 |
| 护甲 (DEF) | 减伤百分比 |
| 拾取范围 (RANGE) | 自动拾取经验/道具的范围 |

### 2.3 武器系统

初期设计 4 种基础武器：

| 武器 | 攻击方式 | 特点 |
|------|----------|------|
| 旋转刀刃 | 环绕角色旋转 | 近战 AOE |
| 闪电链 | 自动弹射最近敌人 | 连锁伤害 |
| 火球术 | 向最近敌人发射火球 | 远程单体 |
| 冰霜新星 | 周期性释放冰霜波 | 减速 + AOE |

### 2.4 技能/升级系统

每次升级提供 3 个随机选项：
- **武器升级**：提升已有武器等级（伤害/范围/数量）
- **新武器**：获得新的武器类型
- **被动技能**：增加属性（攻速、移速、暴击等）
- **辅助技能**：护盾、回血、磁铁等

### 2.5 敌人系统

| 敌人类型 | 行为模式 | 特点 |
|----------|----------|------|
| 小怪（近战） | 直线冲向玩家 | 数量多，血少 |
| 小怪（远程） | 保持距离，发射弹幕 | 需要躲避 |
| 精英怪 | 追踪玩家，有特殊技能 | 血多，掉落好 |
| Boss | 固定时间出现，多阶段 | 关卡核心挑战 |

### 2.6 关卡系统

- 每关持续时间：3-5 分钟
- 难度递增：怪物数量/血量/速度逐渐增加
- 关卡结构：3 波小怪 → 精英波 → Boss
- 章节制：每章 10 关，不同主题/场景

## 3. 系统功能

### 3.1 养成系统（Meta Game）

| 系统 | 说明 |
|------|------|
| 角色解锁 | 不同角色有不同初始武器和被动 |
| 装备系统 | 武器/防具/饰品，提供属性加成 |
| 天赋树 | 永久属性提升，用金币解锁 |
| 成就系统 | 完成特定条件获得奖励 |

### 3.2 经济系统

| 货币 | 获取方式 | 用途 |
|------|----------|------|
| 金币 | 关卡掉落、每日任务 | 升级天赋、购买装备 |
| 钻石 | 成就、广告、付费 | 抽取角色/武器 |

### 3.3 社交系统（后期）

- 排行榜（关卡通关时间、击杀数）
- 好友系统
- 每日挑战

## 4. 非功能性需求

### 4.1 性能要求

| 指标 | 目标值 |
|------|--------|
| 帧率 | 稳定 60 FPS |
| 同屏敌人数 | ≥ 100 个 |
| 启动时间 | < 3 秒（热启动） |
| 包体大小 | < 100MB（初始包） |
| 内存占用 | < 300MB |

### 4.2 兼容性

- iOS 12+ / Android 7.0+
- 最低支持机型：iPhone 8 / 骁龙 660 级别

### 4.3 可维护性

- 模块化架构，方便热更新
- 数据驱动设计（关卡/怪物/技能配置表化）
- ECS 或组件化架构

## 5. 里程碑规划

| 阶段 | 时间 | 交付物 |
|------|------|--------|
| **MVP** | 2 周 | 角色移动 + 1种武器 + 基础怪物 + 升级系统 |
| **Alpha** | 4 周 | 4种武器 + 技能系统 + 3个关卡 + Boss |
| **Beta** | 6 周 | 养成系统 + 装备 + 10个关卡 + UI完善 |
| **Release** | 8 周 | 性能优化 + 音效 + 引导 + 商业化 |

## 6. 风险识别

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 同屏大量敌人性能问题 | 卡顿 | 对象池 + GPU实例化 + LOD |
| 技能数值平衡 | 体验差 | 数据配置表 + A/B测试 |
| 包体过大 | 用户流失 | 资源分包 + 动态加载 |
| Cocos 生态插件不足 | 开发慢 | 提前调研，必要时自研 |
