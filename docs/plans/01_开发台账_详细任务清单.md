# 割草游戏（草原风暴）- 详细开发台账任务清单

> 本文档囊括项目全部 **46 个开发任务**，按模块和优先级排列。
> 每个任务包含：开发目标、开发顺序、依赖关系、涉及页面、涉及接口、涉及数据对象、验收标准、测试与验证方式。

---

## 阅读说明

- **任务编号格式**：`M{模块号}-T{序号}`，如 M01-T01
- **优先级**：P0（阻塞性）> P1（核心功能）> P2（重要）> P3（锦上添花）
- **状态**：🔲 未开始 | 🔨 进行中 | ✅ 完成 | ⏸ 阻塞

## M01 - 核心框架模块 (Core)

### M01-T01 事件总线 (EventBus)

| 项目 | 内容 |
|------|------|
| **开发目标** | 实现全局事件总线，支持 on/off/once/emit，模块间通过事件解耦通信 |
| **开发顺序** | 第 1 位（所有模块基础） |
| **依赖关系** | 无前置依赖 |
| **涉及页面** | 无（纯逻辑层） |
| **涉及接口** | 定义 GameEvent 枚举（20+ 事件类型） |
| **涉及数据对象** | EventEntry {callback, target, once} |
| **验收标准** | ① 支持注册/注销/触发事件 ② 支持一次性监听(once) ③ 支持按 target 批量注销 ④ 无内存泄漏 |
| **测试与验证** | 单元测试：注册事件→触发→验证回调执行；注销后触发→验证不执行；once 只执行一次 |
| **状态** | ✅ 已完成 |

| 项目 | 内容 |
|------|------|
| **开发目标** | 实现通用 Node 对象池，支持预加载、获取、归还，避免运行时 GC |
| **开发顺序** | 第 2 位 |

| **涉及数据对象** | pools: Map<string, Node[]>, activeCount: Map<string, number> |
| **验收标准** | ① 预加载100个对象无报错 ② get/put 循环1000次无内存增长 ③ 池为空时自动创建新实例 ④ clearAll 正确销毁所有节点 |
| **测试与验证** | 性能测试：预加载200个→连续get/put 500次→Profile内存无增长；功能测试：池空时get返回新实例 |
| **状态** | ✅ 已完成 |
